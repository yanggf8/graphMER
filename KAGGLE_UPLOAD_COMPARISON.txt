OPTION 1: WEB UPLOAD (Recommended for First Time)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Setup Time:     0 minutes
Total Time:     5 minutes
Difficulty:     ⭐ Easy

Steps:
1. Build package (2 min)
   cd /home/yanggf/a/graphMER
   ./scripts/deploy_to_kaggle.sh
   # Creates: graphmer_kaggle_dataset.zip (~3-5 MB)

2. Upload via web (3 min)
   • Go to: https://www.kaggle.com/datasets
   • Click: "New Dataset"
   • Drag & drop: graphmer_kaggle_dataset.zip
   • Title: "GraphMER Knowledge Graph"
   • Click: "Create"

Pros:
✅ No setup required
✅ Visual interface
✅ Immediate feedback
✅ Easy to verify files
✅ Perfect for beginners

Cons:
❌ 500 MB file size limit (you're only 3-5 MB, so OK)
❌ Manual process (can't automate)

OPTION 2: CLI UPLOAD (Recommended for Frequent Updates)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Setup Time:     5 minutes (one-time)
Total Time:     8 minutes (first time), 2 minutes (updates)
Difficulty:     ⭐⭐ Moderate

Steps:
1. Get API token (2 min, one-time)
   • Go to: https://www.kaggle.com/YOUR_USERNAME/account
   • Scroll to: "API" section
   • Click: "Create New API Token"
   • Download: kaggle.json

2. Setup CLI (1 min, one-time)
   mkdir -p ~/.kaggle
   mv ~/Downloads/kaggle.json ~/.kaggle/
   chmod 600 ~/.kaggle/kaggle.json

3. Build package (2 min)
   cd /home/yanggf/a/graphMER
   ./scripts/deploy_to_kaggle.sh

4. Edit metadata (1 min, one-time)
   cd kaggle_deploy
   nano dataset-metadata.json
   # Change: "id": "YOUR_USERNAME/graphmer-kg"
   # Replace YOUR_USERNAME with your Kaggle username

5. Upload (2 min)
   kaggle datasets create -p .

Pros:
✅ 2 GB file size limit (much larger than web)
✅ Automation ready
✅ Easy to update: kaggle datasets version -p .
✅ Scriptable (for CI/CD)
✅ No browser required

Cons:
❌ Initial setup required
❌ Command line interface
❌ Need to manage API token
